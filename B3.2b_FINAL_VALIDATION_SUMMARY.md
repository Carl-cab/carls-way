## Key Facts for Production

### What Phase B3.2b Does:

✅ Updates wallet balances (balance_usd, balance_cad)  
✅ Tracks balance processing state (balance_processed_at)  
✅ Tracks balance errors (balance_processing_error)  
✅ Prevents duplicate balance updates (idempotent)  
✅ Skips unsupported transfers (cash_out)  
✅ Reverses returned transfers (only if prior balance exists)  

### What Phase B3.2b Does NOT Do:

❌ Create ledger entries (deferred to B3.2a)  
❌ Send notifications (deferred to B3.3)  
❌ Reverse velocity (deferred to B3.3)  
❌ Call provider APIs (sandbox only)  
❌ Execute real transfers (requires B3.4)  

### Safe Scope:

- Only touches `users` and `provider_webhook_events` tables
- No schema changes required
- Rollback is safe (no dependencies)
- Can be disabled via withdrawal of trust without data loss
