# Phase B3.2b Balance Executor — Safe Validation Setup

## ⚠️ CRITICAL: Non-Production Environment Only

This validation setup MUST run in one of these environments:
- **Vercel Preview** (recommended) — isolated database per preview deployment
- **Local development** — local PostgreSQL with separate test database
- **Staging database clone** — copy of production schema, zero production data

**DO NOT RUN IN PRODUCTION** — these tests will create real database mutations.

---

## Part 1: Environment Setup

### Option A: Vercel Preview (Recommended)

**Why:** Each preview deployment gets isolated database; no risk to production.

1. **Create preview branch locally:**
   ```bash
   git checkout -b test/b3.2b-validation
   git push -u origin test/b3.2b-validation
   ```

2. **Vercel auto-deploys preview:**
   - Navigate to https://vercel.com → carl-cab/carls-way → Deployments
   - Find deployment for `test/b3.2b-validation` branch
   - Note preview URL: `https://<preview-hash>.carloscab74.vercel.app`

3. **Verify isolated database:**
   - Vercel dashboard → Settings → Environment Variables
   - Confirm DATABASE_URL for this preview points to **separate Supabase branch** or test database
   - If not, create a `.env.local.test` file with test DATABASE_URL

4. **Confirm non-production:**
   ```bash
   # From preview environment via Vercel dashboard:
   # Check database connection string does NOT contain:
   # - "production" keyword
   # - Same host as carloscab74.vercel.app production
   # - Same schema as live users/transactions
   ```

### Option B: Local Development

1. **Start local PostgreSQL:**
   ```bash
   # macOS (if using Homebrew):
   brew services start postgresql

   # Or Docker:
   docker run -d \
     --name postgres-test \
     -e POSTGRES_PASSWORD=test \
     -e POSTGRES_DB=manna_test \
     -p 5432:5432 \
     postgres:16
   ```

2. **Create test database connection string:**
   ```bash
   export DATABASE_URL="postgresql://postgres:test@localhost:5432/manna_test"
   ```

3. **Initialize test schema:**
   ```bash
   npm run build  # TypeScript builds with test DATABASE_URL
   npm run dev &  # Starts local server, calls initializeSchema() on first boot
   ```

4. **Confirm database is initialized:**
   ```bash
   psql postgresql://postgres:test@localhost:5432/manna_test -c "SELECT tablename FROM pg_tables WHERE schemaname='public';" | head -10
   ```
   Expected output: `users`, `transactions`, `transfer_intents`, `provider_webhook_events`, etc.

### Option C: Staging Database Clone

1. **Request database dump from production:**
   ```bash
   # Supabase dashboard → Database → Backups
   # Export schema only (no data):
   pg_dump -s --no-owner postgresql://prod-url > schema.sql
   ```

2. **Restore to staging database:**
   ```bash
   psql postgresql://staging-url < schema.sql
   ```

3. **Set DATABASE_URL:**
   ```bash
   export DATABASE_URL="postgresql://staging-url"
   ```

4. **Confirm empty of user data:**
   ```bash
   psql $DATABASE_URL -c "SELECT COUNT(*) FROM users;"
   # Expected: 0
   ```

### Test Harness: Curl Requests for Each Scenario

#### Scenario 1: Idempotency via balance_processed_at

```bash
# BEFORE: Record balance
BEFORE_S1=$(psql $DATABASE_URL -c "SELECT balance_usd FROM users WHERE email='test-us-001@validation.local';")
echo "Scenario 1 BEFORE: balance_usd=$BEFORE_S1"

# Simulate first settled webhook
curl -X POST http://localhost:3000/api/webhooks/plaid \
  -H "Content-Type: application/json" \
  -H "Plaid-Verification: <valid-jwt>" \
  -d '{
    "webhook_type": "TRANSFER",
    "webhook_code": "STATUS_UPDATE",
    "data": {
      "transfer_id": "transfer_id_001",
      "status": "settled"
    }
  }' | jq '.'

# AFTER FIRST: Check balance increased
AFTER_FIRST=$(psql $DATABASE_URL -c "SELECT balance_usd FROM users WHERE email='test-us-001@validation.local';")
echo "Scenario 1 AFTER FIRST: balance_usd=$AFTER_FIRST (expected: 5100)"

# Verify balance_processed_at set
psql $DATABASE_URL -c "SELECT provider_event_id, balance_processed_at FROM provider_webhook_events WHERE provider_event_id='evt-001-first';"

# Replay same webhook (due to UNIQUE constraint, treated as duplicate at intake)
# Expected: Returns 200 with duplicate: true
echo "Attempting replay (should be rejected at intake due to UNIQUE constraint)..."

# AFTER REPLAY: Check balance unchanged
AFTER_REPLAY=$(psql $DATABASE_URL -c "SELECT balance_usd FROM users WHERE email='test-us-001@validation.local';")
echo "Scenario 1 AFTER REPLAY: balance_usd=$AFTER_REPLAY (expected: 5100, unchanged)"

# RESULT: 
if [ "$AFTER_FIRST" = "5100" ] && [ "$AFTER_REPLAY" = "5100" ]; then
  echo "✅ Scenario 1 PASS: Idempotency verified"
else
  echo "❌ Scenario 1 FAIL: Idempotency not verified"
fi
```
